### 1. Veri Tipi Önekleri (Prefixes)

Veri tiplerinin başındaki kısaltmalar veritabanının veriyi nasıl sakladığını belirler:

- **`N` (National - Örn: NVARCHAR):** Unicode karakter setini destekler. Çince, Arapça, Kiril alfabesi veya Emoji (😊) gibi özel karakterleri saklayabilirsin. Her karakter 2 byte yer kaplar.
    
- **`VAR` (Variable - Örn: VARCHAR):** "Değişken uzunluk" anlamına gelir. `VARCHAR(50)` tanımlayıp içine "Ali" (3 karakter) yazarsan, hafızada 50 değil, sadece 3 karakterlik yer kaplar. Depolama tasarrufu sağlar.

### 2. DML (Data Manipulation Language) Komutları
Veri üzerinde işlem (ekleme, silme, güncelleme) yapan komutlardır.

#### **SELECT** (Listeleme)
Veritabanından veri çekmek için kullanılır.
```
SELECT Kolon1, Kolon2 
FROM TabloAdi;
```

#### **INSERT** (Ekleme)
Tabloya yeni satır ekler.
```
INSERT INTO TabloAdi (Kolon1, Kolon2) 
VALUES ('Değer1', 'Değer2');
```

#### **UPDATE** (Güncelleme)
Mevcut veriyi değiştirir.
> ⚠️ **Dikkat:** `WHERE` koşulu yazmazsan tablodaki **TÜM** kayıtları güncellersin!
```
UPDATE TabloAdi
SET Kolon1 = 'YeniDeger', Kolon2 = 'BaskaDeger'
WHERE ID = 5;

-- Örnek (Yaş Hesaplama):
UPDATE CUSTOMERS 
SET AGE = DATEDIFF(YEAR, BIRTHDATE, GETDATE());
```

#### **DELETE** (Silme)
Satır bazlı silme işlemi yapar. Geri alınabilir (Transaction içindeyse).
> ⚠️ **Dikkat:** `WHERE` koşulu yazmazsan tablodaki **TÜM** veriler silinir!
```
DELETE FROM TabloAdi 
WHERE Sehir = 'Ankara';
```

#### **TRUNCATE** (Sıfırlama)
Tablonun içini tamamen boşaltır.
- `DELETE`'ten çok daha hızlıdır.
- Tablo yapısını (kolonları) silmez, sadece veriyi uçurur.
- Identity (ID) sayacını sıfırlar.
```
TRUNCATE TABLE TabloAdi;
```

### 3. Filtreleme ve Şartlar (`WHERE`)
Verileri filtrelemek için kullanılır.

- **Eşit Değildir (`<>`):**
    ```
    WHERE AGE <> 18 -- Yaşı 18 olmayanlar
    ```
    
- **BETWEEN (Arasında):**
    ```
    WHERE AGE BETWEEN 20 AND 40 -- 20 ve 40 dahil
    ```
    
- **LIKE (Benzerlik):** Metin içinde arama yapar. `%` joker karakterdir.
    ```
    WHERE NAMESURNAME LIKE 'Ali%'  -- Ali ile başlayanlar
    WHERE NAMESURNAME LIKE '%Ali'  -- Ali ile bitenler
    WHERE NAMESURNAME LIKE '%Ali%' -- İçinde Ali geçenler
    ```
    
- **IN (Liste İçinde):** Birden fazla `OR` yazmak yerine kullanılır.
    ```
    WHERE AGE IN (23, 33, 43) -- Yaşı 23, 33 veya 43 olanlar
    ```
    
- **AND / OR (Mantıksal Operatörler):**
    ```
    WHERE CITY = 'Istanbul' AND (GENDER = 'E' OR AGE > 25)
    ```

### 4. Sonuç Şekillendirme Komutları

#### **ORDER BY** (Sıralama)
Sorgu sonucunu belirli bir düzene göre sıralar.
- **ASC (Ascending):** A'dan Z'ye veya Küçükten Büyüğe (Varsayılan).
- **DESC (Descending):** Z'den A'ya veya Büyükten Küçüğe.

```
SELECT * FROM CUSTOMERS
ORDER BY CITY ASC, AGE DESC; -- Şehre göre A-Z, aynı şehirde olanları Yaşa göre tersten sırala.
```

#### **DISTINCT** (Tekilleştirme)
Tekrar eden satırları eler, sadece benzersiz kayıtları getirir.
```
SELECT DISTINCT CITY FROM CUSTOMERS; -- Sadece farklı şehir isimlerini getirir.
```

#### **TOP** (Limit - SQL Server)
Dönen sonucun sadece ilk N tanesini getirir. (MySQL ve PostgreSQL'de `LIMIT` kullanılır).
```
SELECT TOP 100 * FROM CUSTOMERS
ORDER BY ID DESC; -- Son eklenen 100 kaydı getir.
```
~~~
SELECT TOP 10 PERCENT * FROM CUSTOMERS -- PERCENT İLE YÜZDE 10LUK DİLİMİ GETİRİR
~~~
#### **ALIAS - AS** (Takma Ad)
Tablo veya kolonlara geçici isim vermek için kullanılır. Kodun okunabilirliğini artırır.
```
SELECT 
    C.NAMESURNAME AS [Müşteri Adı], 
    C.TELNR AS [Telefon]
FROM CUSTOMERS AS C; -- AS opsiyoneldir, yazılmasa da çalışır.
```


### 5. DDL (Data Definition Language) Komutları
DDL komutları veritabanındaki verilerle (satırlarla) değil, **veritabanı nesnelerinin yapısıyla** (tablolar, view'lar, index'ler) ilgilenir.

#### **CREATE** (Oluşturma)
Yeni bir veritabanı nesnesi (Tablo, View, Stored Procedure vb.) oluşturmak için kullanılır.

**Örnek Senaryo (Müşteri Tablosu Oluşturma):**
```
CREATE TABLE CUSTOMERS (
    ID INT IDENTITY(1,1) PRIMARY KEY, -- 1'den başla, 1'er artır. Benzersiz (PK) olsun.
    NAMESURNAME VARCHAR(100) NOT NULL, -- Boş geçilemez.
    GENDER VARCHAR(1),
    BIRTHDATE DATE,
    CITY VARCHAR(50),
    TOWN VARCHAR(50),
    TELNR VARCHAR(15),
    TCNO VARCHAR(11)
);
```

> **Senior Notu:** `IDENTITY(1,1)` ifadesi, her yeni kayıtta ID'yi otomatik verir. Sen insert atarken ID yazmazsın.

---

#### **ALTER** (Değiştirme/Düzenleme)
Var olan bir nesnenin yapısında değişiklik yapmak için kullanılır. Tabloyu silmeden kolon ekleyip çıkarabilirsin.

**1. Yeni Kolon Ekleme (`ADD`)** Tabloya sonradan yeni bir sütun eklemek için:
```
ALTER TABLE CUSTOMERS 
ADD AGE INT; 

ALTER TABLE CUSTOMERS 
ADD AGEGROUP VARCHAR(20);
```

**2. Kolon Özelliklerini Değiştirme (`ALTER COLUMN`)** Bir kolonun veri tipini veya `NULL/NOT NULL` durumunu değiştirmek için:
```
-- GENDER kolonunu 1 karakterden 10 karaktere çıkarıyoruz
ALTER TABLE CUSTOMERS 
ALTER COLUMN GENDER VARCHAR(10);

-- İsim soyisimi zorunlu (NOT NULL) hale getiriyoruz
ALTER TABLE CUSTOMERS 
ALTER COLUMN NAMESURNAME VARCHAR(100) NOT NULL;
```

**3. Kolon Silme (`DROP COLUMN`)** Tablodan bir sütunu tamamen kaldırmak için:
```
ALTER TABLE CUSTOMERS 
DROP COLUMN AGEGROUP;
```

**4. Primary Key Ekleme** Eğer tabloyu yaratırken PK vermeyi unuttuysan sonradan ekleyebilirsin:
```
ALTER TABLE CUSTOMERS 
ADD PRIMARY KEY (ID);
```

---
#### **DROP** (Silme/Yok Etme)
Veritabanı nesnesini (tabloyu, veritabanını vb.) tamamen siler.
> ⚠️ **Çok Kritik Uyarı:** `DROP TABLE` komutu tabloyu içindeki verilerle beraber tamamen yok eder. Geri dönüşü yoktur!
```
DROP TABLE CUSTOMERS;
```
`DELETE`, `TRUNCATE` ve `DROP` arasındaki fark nedir?

- **DELETE (DML):** Satırları siler (Örn: Sadece Ahmet'i sil). Tablo kalır. Yavaştır, log tutar.
- **TRUNCATE (DDL):** Tablonun içini tamamen boşaltır. Tablo iskeleti kalır. Çok hızlıdır, ID'yi sıfırlar.
- **DROP (DDL):** Tabloyu kökten yok eder. Ne veri kalır ne tablo.
