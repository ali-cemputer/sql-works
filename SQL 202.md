### 1. Aggregate Functions (Kümeleme Fonksiyonları)
Bu fonksiyonlar, dikeydeki satırları alıp matematiksel işlemlerle tek bir sonuca (özete) dönüştürür.

|**Fonksiyon**|**Açıklama**|**Örnek Kullanım**|
|---|---|---|
|**SUM**|Değerlerin toplamını hesaplar.|`SUM(AMOUNT)`|
|**MIN**|En küçük değeri bulur.|`MIN(PRICE)`|
|**MAX**|En büyük değeri bulur.|`MAX(PRICE)`|
|**AVG**|Ortalamayı hesaplar.|`AVG(AGE)`|
|**COUNT**|Satır sayısını (adet) verir.|`COUNT(*)` veya `COUNT(ID)`|

#### **Temel Sözdizimi (Syntax)**

```
SELECT
    SUM(AMOUNT) AS ToplamTutar,
    AVG(PRICE) AS OrtalamaFiyat,
    MIN(DATE) AS IlkTarih,
    MAX(DATE) AS SonTarih,
    COUNT(*) AS ToplamIslemSayisi
FROM SALES
WHERE DATE > '2023-01-01'; -- Şartlar buraya
```

#### **Tekilleştirerek Sayma (Distinct Count)**
Bir kolondaki tekrar eden verileri 1 kez saymak için kullanılır.
```
-- Kaç farklı şehre satış yaptık?
SELECT COUNT(DISTINCT CITY) FROM SALES;
```

### 2. GROUP BY (Gruplama)
Aggregate fonksiyonları kategorik bir veriye (Şehir, Cinsiyet, Yıl vb.) göre parçalamak için kullanılır.

> ⚠️ **Altın Kural:** `SELECT` kısmında aggregate fonksiyon (SUM, AVG vb.) **dışında** kalan tüm kolonlar, `GROUP BY` kısmında **yazılmak zorundadır.**

```
SELECT
    CITY,               -- 1. Gruplamak istediğim değişken
    SUM(AMOUNT) AS ToplamSatis,
    COUNT(*) AS IslemAdedi
FROM SALES
WHERE DATE >= '2024-01-01'
GROUP BY CITY           -- 2. Aynı değişkeni buraya da yazmalısın
ORDER BY ToplamSatis DESC;
```

---
### 3. HAVING (Gruplanmış Veriyi Filtreleme)
`WHERE` komutu satırları gruplamadan **önce** filtreler. `HAVING` ise gruplama yapıldıktan ve hesaplamalar bittikten **sonra** çıkan sonucu filtreler.

> **Kısaca:** Aggregate fonksiyonlara (`SUM`, `COUNT` vb.) şart vereceksen `HAVING` kullanmalısın.

```
SELECT
    CITY,
    SUM(AMOUNT) AS ToplamCiro,
    AVG(AMOUNT) AS OrtalamaSatis
FROM SALES
WHERE GENDER = 'E'          -- 1. ÖNCE: Sadece Erkek müşterileri al (Satır Bazlı Filtre)
GROUP BY CITY               -- 2. SONRA: Şehre göre grupla
HAVING SUM(AMOUNT) > 10000  -- 3. EN SON: Cirosu 10.000'den büyük olan şehirleri getir (Grup Bazlı Filtre)
ORDER BY CITY;
```

### 🧠 Senior Notu: WHERE vs HAVING Farkı
Mülakatlarda en çok sorulan sorulardan biridir:

| **Özellik**             | **WHERE**                                   | **HAVING**                                  |
| ----------------------- | ------------------------------------------- | ------------------------------------------- |
| **Çalışma Zamanı**      | Gruplamadan **ÖNCE** çalışır.               | Gruplamadan **SONRA** çalışır.              |
| **Kullanım Amacı**      | Ham veriyi (satırları) filtreler.           | Hesaplanmış veriyi (SUM, AVG) filtreler.    |
| **Aggregate Kullanımı** | İçinde `SUM()`, `COUNT()` **kullanılamaz.** | İçinde `SUM()`, `COUNT()` kullanılabilir.   |
| **Performans**          | Daha hızlıdır (Veriyi baştan eler).         | Daha yavaştır (Hesaplama yapıp sonra eler). |

### 4. İlişkisel Veritabanı ve JOIN İşlemleri

İlişkisel veritabanı (RDBMS), verilerin tekrar etmesini önlemek (tekilleştirmek) ve veri bütünlüğünü sağlamak için verileri farklı tablolara bölen sistemdir. Bu tablolar arasındaki bağı kurmak için **JOIN** işlemi kullanılır.

#### Temel JOIN Sözdizimi (Syntax)
Tablolara **Alias (Takma Ad)** vermek (`A`, `B`, `U` gibi) sorgu yazmayı hızlandırır ve okunabilirliği artırır.
```
SELECT 
    A.Kolon1, 
    B.Kolon2
FROM Tablo1 AS A
JOIN Tablo2 AS B ON A.OrtakKolonID = B.OrtakKolonID;
```

---
### JOIN Türleri (Venn Şeması Mantığı)
Verileri kümeler teorisi (Venn şeması) gibi düşünmelisin.

#### 1. INNER JOIN (Kesişim)
Sadece **her iki tabloda da eşleşen** kayıtları getirir. Ortak olmayanlar gelmez.
- **Mantık:** $A \cap B$ (Sadece ortak alan)

```
-- Sadece adresi tanımlı olan kullanıcıları getir
SELECT 
    U.USERNAME_,
    A.CITY
FROM USERS AS U
INNER JOIN ADDRESS AS A ON U.ID = A.USERID;
```

#### 2. LEFT JOIN (Sol + Kesişim)
Sorgunun **SOLUNDAKİ** tablonun tamamını getirir. Sağdaki tabloda karşılığı varsa onu da getirir, yoksa `NULL` yazar.
- **Mantık:** Solun hepsi, Sağın eşleşenleri.
- **Kullanım:** "Adresi olmayan kullanıcıları bul" demek için idealdir.

```
-- Tüm kullanıcıları getir, adresi varsa yaz yoksa NULL geç
SELECT 
    U.USERNAME_,
    A.CITY
FROM USERS AS U
LEFT JOIN ADDRESS AS A ON U.ID = A.USERID;
```

#### 3. RIGHT JOIN (Sağ + Kesişim)
Sorgunun **SAĞINDAKI** tablonun tamamını getirir. Soldaki tabloda karşılığı varsa getirir.
- **Mantık:** Sağın hepsi, Solun eşleşenleri.
- **Senior Notu:** Pratikte çok az kullanılır. Genellikle tabloların yeri değiştirilip `LEFT JOIN` kullanılır (Okunabilirlik açısından soldan sağa okumak daha kolaydır).

```
SELECT *
FROM USERS AS U
RIGHT JOIN ADDRESS AS A ON U.ID = A.USERID;
```

#### 4. FULL OUTER JOIN (Birleşim)
İki tablodaki **TÜM** kayıtları getirir. Eşleşenleri eşleştirir, eşleşmeyen tarafları `NULL` bırakır.
- **Mantık:** $A \cup B$ (Hepsi)

```
SELECT 
    U.USERNAME_, 
    A.CITY
FROM USERS AS U
FULL OUTER JOIN ADDRESS AS A ON U.ID = A.USERID;
```

### ⛓️ Zincirleme JOIN (Multiple Joins)
Gerçek hayatta genelde 2'den fazla tabloyu birbirine bağlarız. Zincirleme mantığı kullanılır.
**Senaryo:** Müşteri (`C`), Adres (`A`) ve Şehir (`CT`) tablolarını birleştirme.
```
SELECT 
    A.POSTALCODE, 
    A.ADDRESSTEXT,
    CT.CITY,           -- Cities tablosundan şehir adı
    C.NAMESURNAME,     -- Customers tablosundan isim
    C.EMAIL
FROM ADDRESS AS A
JOIN CUSTOMERS AS C ON C.ID = A.CUSTOMERID  -- Adresi Müşteriye bağla
JOIN CITIES AS CT ON CT.ID = A.CITYID       -- Adresi Şehre bağla
```

### 💾 Bonus: Tablo Yedekleme (SELECT INTO)
Mevcut bir tablonun yapısını ve verisini kopyalayarak **yeni bir tablo** oluşturur.
> ⚠️ **Uyarı:** Bu komut tabloyu sıfırdan yaratır. Primary Key, Index gibi yapısal özellikleri kopyalamaz, sadece veriyi ve sütun tiplerini kopyalar.
```
-- Adres tablosunun yedeğini ADRESSYEDEK adıyla oluştur
SELECT * INTO ADRESSYEDEK 
FROM ADDRESS;
```
